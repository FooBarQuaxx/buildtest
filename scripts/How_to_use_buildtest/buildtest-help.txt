usage: buildtest [options]

optional arguments:
  -h, --help            show this help message and exit
  -V, --version         show program version number and exit

Basic Options:
  buildtest basic options

  --check-setup         Check buildtest configuration and determine if you
                        have it setup properly for testing
  --module-naming-scheme {HMNS,FNS}
                        Specify module naming scheme for easybuild apps
  --scantest            Report all tests that can be built with buildtest by
                        checking all available apps found in eb stack and
                        system packages

Find Options:
  buildtest options for finding software, toolchains, tests, yaml files

  -lt, --list-toolchain
                        retrieve toolchain used based on the easyconfig files
                        provided by BUILDTEST_EASYCONFIGDIR
  -ls, --list-unique-software
                        retrieve all unique software found in your module tree
                        specified by BUILDTEST_MODULETREE
  -svr, --software-version-relation
                        retrieve a relationship between software and version
                        found in module files
  -fc FINDCONFIG, --findconfig FINDCONFIG
                        Find buildtest YAML config files found in
                        BUILDTEST_SOURCEDIR. To find all yaml config files use
                        -fc all
  -ft FINDTEST, --findtest FINDTEST
                        Find test scripts generated by buildtest defined by
                        BUILDTEST_TESTDIR. To find all test scripts use -ft
                        all

Test Options:
  Options for building tests with buildtest

  -s INSTALLED-EASYBUILD-APPS, --software INSTALLED-EASYBUILD-APPS
                        Specify software package to test
  -t INSTALLED-EASYBUILD-TOOLCHAINS, --toolchain INSTALLED-EASYBUILD-TOOLCHAINS
                        Specify toolchain for the software package
  --shell {sh,bash,csh}
                        Select the type of shell when running test
  --system SYSTEM-PACKAGE
                        Build test for system packages To build all system
                        package test use --system all
  --testset {Python,R,Ruby,Perl,Tcl,MPI}
                        Select the type of test set to run (Python, R, Ruby,
                        Perl, Tcl, MPI)

YAML Options:
  Options for YAML configuration

  --sysyaml INSTALLED-SYSTEM-PACKAGE
                        generate binary test YAML configuration for system
                        package
  --ebyaml EBYAML       generate binary test YAML configuration for easybuild
                        package (Not Implemented)

Job Scheduler Options:
  Options for interacting with Job Scheduler

  --job-template JOB_TEMPLATE
                        specify job template file to create job submission
                        script for the test to run with resource scheduler
  --submitjob SUBMITJOB
                        specify a directory or job script to submit to
                        resource scheduler

Miscellaneous Options:
  rest of buildtest options

  --runtest             Run the test interactively through runtest.py
